The product

A checkout layer for AI agents buying paid APIs and RWA data.

Agent commerce needs more than wallets. API sellers need a way to charge agents per request, and buyers need policy controls plus public receipts before autonomous systems can spend real value.

01

Merchant API console

Publish a paid RWA API with price, endpoint hash, service id, and a machine-readable payment requirement.

02

MCP buyer agent

The agent requests the paid API through tools, but policy gates amount, service allowlist, budget, approvals, and replay.

03

Casper receipt layer

Approved agent purchases are committed through a deployed Odra ReceiptLedger contract on Casper Testnet.

AgentPay is not a generic AI dashboard. The checkout flow is designed around x402-style API payment, MCP tool access, Odra contracts, Testnet receipt proof, and the CSPR.cloud/CSPR.click final-round path.

System design

Built around the Casper AI Toolkit primitives.

x402-style payments

The merchant endpoint responds with payment-required terms that an agent can satisfy only after policy approval.

Policy engine

Deterministic checks prevent overspend, unapproved services, expired policies, duplicate actions, and unsafe approvals.

MCP gateway

JSON-RPC tool surface gives buyer agents controlled access to quote, simulate, execute, and inspect proof.

Casper contract

Odra ReceiptLedger records approved commerce receipts and exposes readback state for judges and users.
