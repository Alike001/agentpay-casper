# Story 02 - Dashboard and Wallet

Goal: Build the user dashboard for wallet connection, agent creation, policy editing, and proof display.

Constraints:

- Use a restrained operational UI.
- No marketing landing page as first screen.
- Keep text readable on mobile and desktop.

Acceptance:

- User can connect a Casper wallet.
- User can view/create an agent.
- User can view/edit policy fields.
- User can see transaction proof and receipt timeline.
- Empty, loading, success, and error states exist.

UI surfaces:

- Left nav: Agents, Services, Receipts, Settings.
- Main dashboard: active agent, policy limits, live activity.
- Policy editor: spend cap, allowed services, approval threshold, expiry.
- Receipt timeline: allowed, blocked, approval-required states.
